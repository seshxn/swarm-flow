# Vision

swarm-flow exists to make AI-assisted software delivery inspectable, resumable, and governable.

The project is not trying to be the smartest autonomous engineer. It is trying to be the cleanest delivery engine for teams that want AI agents to help across the SDLC without losing control over scope, evidence, approvals, or external writes.

## Product Lane

swarm-flow owns delivery orchestration:

- flows define what happens next
- skills define how a phase is executed well
- hooks define automation around transitions
- agents perform the work
- connectors read and write external tools
- policies decide whether execution may continue
- artifacts prove progress

## Principles

- Flows over prompts
- Artifacts over chat logs
- Specialists over generalists
- Evidence over confidence
- Governance over unchecked autonomy
- Resumability over one-shot execution
- Hooks around every meaningful transition
