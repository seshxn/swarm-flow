# MVP Scope

## v0.1 Includes

Given a feature idea, swarm-flow should be able to:

- gather local repo context
- produce a feature brief
- produce acceptance criteria
- produce a technical design
- produce an ADR
- produce a task plan
- preview Jira and Confluence changes
- create a branch
- prepare implementation work through artifacts and examples
- run validation commands
- produce review and QA artifacts
- draft a PR summary and release notes

## v0.1 Excludes

- auto deploy to production
- autonomous merge to main
- full backlog grooming
- cross-team program management
- stakeholder messaging automation
- free-form multi-hour autonomous loops without checkpoints
